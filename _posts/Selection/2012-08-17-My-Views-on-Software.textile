近來覺得自己的軟件觀受到懷疑，特此整理公示。（話說我發現我對自己觀念方面的東西特別有整理意識。大概是被思想家整慘了？哼哼，軟件使用思想能總結出一篇東西來，也算我的本事。）如果有參考價值，當然更好。

h2. 一、戰略層面：軟件是用來服務我而非折磨我的。換言之，是讓我更高效舒適的，所以我儘量選取省事的方式。

h3. （一）、除非問題得不到解決，不然儘量少嘗試新軟，對於自己熟悉的軟件，如果沒有需要，也不多折騰

很多 geek 願意花大量時間研究某個新發現的軟件。我覺得實在沒意思。只要我在原有條件下很安逸，何必多試？除非很明確知道他有這個那個我確實很希望獲得的功能，不然我不會貿然嘗試。或者，至少他有吸引我的一面，而且我能快速上手。也就是說，我非常尊重自己的用戶習慣和學習曲線。這跟我讀書的方式比較像，我不會亂翻書，而是有目的的翻書。

h3. （二）、不嘗試不代表不瞭解

我還是偶爾上一些軟件測評、介紹的小站看看的，至少能夠在自己需要某軟件的時候，知道什麼軟件好用。同時，因爲我不喜歡折騰，可能有什麼功能比我現有軟件強很多很多的軟件出來了都不知道，讀讀測評、介紹，大致瞭解一下，但不加以嘗試，不費事，但也不閉塞。

h3. （三）、不主動做沒必要的升級

有的人是升級控，一定要用最新的軟件才哈皮。起初我也試過，最後確立了夠用就好的思維，因此系統還是 XP ， Office 還是 2003 ，網盤裡存個 Nero ，也衹存了 6 和 7 。但是，中國很多公司比較惡劣，他提供免費軟件的同時，會強制用戶升級到新版，而且是差一個小版本號就不讓用，非常嚴格。對於這類軟件，就很難說了。還有的軟件不強制升級，但不斷彈窗，基於省事原則，也只好升級。衹有某些開發很積極的軟件，我會欣然升級，因爲升級後往往能看到不少新功能，或至少修正了一些常見 bug 。還有就是有些軟件實在 bug 太多了，出了新版就會忍不住試試看能否少一點……

h3. （四）、與其開發一個軟件的潛能，不如換一個專業軟件

善用佳軟的站長總喜歡在介紹軟件的帖子裡加一段很文青的內容，說自己又用該什麼軟件做了一件你想不到的事，並認爲這就是「善用」了該軟件的表現，因爲他最大限度開發了該軟件的功能，正是因爲他「善用」，所以才成就了該軟件的「佳軟」本質。恰恰相反，這正是不能善用軟件的表現。砍樹明明有斧頭和鋸子，現在還有電鋸，他偏偏用菜刀，甚至勺子。這的確是想像力豐富的表現，很多外國閒人喜歡做這種事，但你能說用勺子砍樹是「善用」勺子嗎？遇到大樹，捨棄勺子，尋找電鋸，才是善用的表現。勺子是吃飯喝湯的「佳」器，電鋸是砍樹「佳」器，倒過來必然是行不通的。
這跟第一條衝突嗎？不衝突。因爲我嘗試的前提是有需求，即，現有軟件離滿意實在差很多。

h3. （五）、大眾化、標準化：用，或至少保留業內評價最高、最熱門、背景最雄厚的軟件

大眾認可總有其合理性，專業軟件尤然。基本上如果不是軟件本身確實優秀，那就是歷史原因形成今日地位，總會有他的可取之處。而且用非主流的東西，往往就是給自己找事：
第一、非主流不便交流。有時想和別人做該類別軟件下的交流，就有一堆兼容性問題。比如系統，雖然我對 Linux 很感興趣，但無意把他作爲主要系統，因爲很多軟件衹有 Windows 客戶端——我對 MAC 當然更加沒興趣， Linux 還說服務器都用他， MAC 就是一群裝逼眾愛用罷了。對於一般軟件，也往往有格式衝突的問題，如果老闆用 Word ，公事上就別用 LaTeX 。
第二、非主流的支持不好。電腦問題，百度 Google 的幫助很大。如果軟件受眾太小，根本沒人寫心得，那就只能孤軍奮戰了。我不是高手，也沒有美國時間。
第三、非主流的可能會出現後續問題。主流作品受眾多，往往能夠保證持續更新，以應對未來出現的一些問題。非主流哪天不更新，然後又出現點什麼新情況，就沒法用了。這當然無法接受。
第四、儘管如此，我還是用了很多非常小眾的軟件。不過這些軟件的功能比較難替代，其實也相當於業界主流了，而且多數是小工具類型的。
第五、對於生成文件的軟件，不放棄使用行業標準。比如生成 PDF ，當然是 Adobe 最權威。所以我一般會保存一個他的虛擬打印機。
第六、使用能夠導出常用格式的軟件。如果 OpenOffice 不能導出 doc ，我相信沒人愛用他老人家。導出功能不僅僅是爲了交流，更是爲以後更換軟件做準備。試想，如果一個軟件，你用來生成了很多文檔，結果有一天發現他不如某某新軟件，而你又無法導出，只好手動貼過去，那是多麼痛苦的一件事。所以能夠導出爲其他軟件可以輕易打開的格式，是非常有意義的事情。

h3. （六）、多用通吃型的軟件

使用通吃軟件，是尊重自己用戶習慣的行爲。軟件和軟件總是有區別的，逐一適應需要過程，所以一個軟件能輕鬆解決，當然最好。這跟第四條並不衝突，那是用勺子砍樹的問題，而我現在討論的是起房子、做乒乓球臺和打狗這三個功能，那板磚是個不錯的選擇。譬如說 UnicornViewer ，我常用的電子書格式他只有 nh 不能讀，自然是比較優越的了。

h3. （七）、分級考慮安全和隱私

當你打字的時候，有沒有想過你打的內容有可能正通過輸入法上傳到某個服務器？當你聊天的時候，有沒有一種好像一位警察叔叔就坐在旁邊看你打字的感覺？當你寫好一篇文章的時候，有沒有擔心文章被人竊取、查看？
有的人可能不在乎，但我是在乎的。我有些自己寫的文章不「合鞋」，這些文章如果被哪個變態軟件搜出來上傳，或者我自己腦殘用同步軟件同步，再或者我同步到國外服務器的過程中被截獲，都是很不好的事情。每個軟件都值得精心篩選啊。

h3. （八）、做好轉換平臺的準備

雖然現在 Windows 讓我很是安逸，可焉知將來會不會是 Linux 的世界？又如何保證我不會改用安卓？不論如何，如果一個軟件支持更多的平臺，對我來說是個好事。

h3. （九）、選擇對自己來說學習曲線平滑且有終點，或者僅僅陡直一段時間就平滑的

比如說 Word ，前期平滑，但簡直沒有終點，即使正常的用了十年廿年，很多他可以實現的功能做不出，這種軟件我不會用作常用軟件。
再如 VI ，是個曲線非常陡直，而且終點遙不可及的編輯器。比如我非常需要，而且非常簡單的自定義顏色，竟然也需要很複雜的設置，想加點小功能就非要到處下載、看 manuals ，這絕對是折磨人的貨。與其用他，不如用破解版做得非常好的或其他免費的。
我喜歡那種能讓你接觸一段時間後就忘了他的存在的軟件。比如說 QQ 拼音，我覺得就做得非常好，免費無廣告，可設置的內容也非常豐富，詞庫多，繁簡轉換函數也是國內一流，經過一番設置，至少我打簡化字時並不覺得是在「用」軟件。除了偶爾因繁簡轉換出現一些問題，簡直比用筆還方便，因此我認爲這是簡化字用戶的最佳選擇。再比如說排版語言 BD ，他有一個簡寫功能，當你把一篇文章的總格式定義好，再自定義好一大堆簡寫的代碼，然後把代碼用自定義短語的方式放進輸入法，在寫作過程中，既不需要像各種 Word 那樣不時點上面粗體一下、下劃線一下，也不需要切換中英，而且標記語言不會像 html 之類的那樣花得有時候很難找到正文。還有像 SpiderOak ，遇到衝突文件，不是像 Dropbox 那樣粗暴而自以爲是的把你電腦上剛編輯的文件肆意重命名，然後把服務器的舊文件下載下來。 SpiderOak 衹是給這文件多了一個版本，如果你覺得服務器上的版本才是所需，再下載就好了。

h3. （十）、不因軟件是否收費而過度影響取捨

雖然各公司都在積極反盜版，不過破解總是強大的，到了我的手裡，絕大多數軟件都有破解版。事實上，對於公司而言，用戶改投別家比用戶繼續用盜版還慘，因爲潛在客戶就這麼流失了。所以 TotalCMD 的作者有他那種寬鬆的態度， WinRAR 其實也可以無限期使用， Nero 出了個閹割過的免費版， MSO 也有免費的 viewer 。現實中這種流失也時有發生，比如方正公司的飛騰，就因爲太難獲得盜版了，用戶衹好用 Adobe InDesign 。後者在處理中文書刊明顯遠弱於飛騰，但是盜版很容易獲得，所以到處都是。現在，小型出版單位、打印店普遍使用盜版 InDesign ，而飛騰衹有大公司在用，結果就是方正怎麼也做不大。好吧，這並非使用盜版的藉口，因爲用盜版就用了，根本不會考慮什麼藉口不藉口。各種虛偽的藉口就好像好色男在理學背景下納妾硬說是爲了傳宗接代一樣，根本沒有任何現實意義。
我選擇軟件還是根據其他「戰略」和「戰術」決定的，免費不免費，對我的影響，大概僅在於破解版的尋求難度。比如方正書版，我用不上尚未破解的 2008 ，就用已完美破解的 10 。再比如各種文件管理軟件， Directory Opus 比較難破解，就不用，用 TotalCMD 。我也不會因爲某款強大的收費軟件容易破解而非要用他，比如 Nero ，我已經很久不碰了，因爲一個小小的 ONES 已經足夠，而且都已經做成綠色版了，還有什麼必要不斷更新？「戰略」是白講的嗎？

h2. 二、戰術層面：延續高效舒適的幾點方案

h3. （一）、性能大於功能

有的軟件功能很多，但很多功能並不需要，或甚少需要，我就選擇安裝性能高的。如果功能多的有什麼功能我偶爾需要，就放一份功能全的在硬盤或硬盤。人很容易被慾望束縛，佔有更多的更能也是人之常情，但這種佔有慾如果不加以克制，其實反而是傷害自身利益的。
比如有時幾兩句話，也有人要開個 Word ，你不覺得很慢、很浪費生命麼？再比如查個字典，有人就非要開個金山詞霸，而捨棄秒開的 MDict 。

h3. （二）、找到合適的下載平臺

我是重度軟件使用者，而且使用大量盜版軟件，很容易碰上病毒。而我又是很少會裝殺毒軟件的，怎麼辦？
官方網站當然是首選，但收費的和有變態廣告的，終究不行。而且我更傾向於使用綠色、便攜的軟件，原版往往不具備該功能，我又比較懶惰。
我會傾向於熱門的審核制論壇式平臺，比如 VeryCD ，雲端。一款軟件做得好不好，有沒有病毒，看看評論就知道。特別像 VeryCD ，資源如果不好，肯定被刪除的。當然他自己因爲版權問題也刪掉了很多，不過盜鏈他的網站也不少，很多一看就知道是 VeryCD 首發的。同時，像 hanzify, PortableApps, PortableAppz, PortableAppC, portablesoft, LiberKey 也都是選項。雖然我下載便攜軟件網站的軟件往往不作爲便攜軟件（因爲覺得沒必要），而作爲普通綠色軟件使用，而且他們的軟件覆蓋面真的很小，不過還是可以看看的。
再然後是不一些版主比較勤快、人氣比較旺的論壇的軟件版上別人推薦的。
如果以上都沒有，就只好冒險試試華彩、多特之類的了。這些就要小心些，壓縮包下下來之後先不解壓，而是打開看看。很多病毒在這個狀態下是不會發作的。
檢驗成果：停用殺毒軟件後一年多，我下載了一個殺毒軟件，最後查到的全是誤報。相比之下，某位軟件都不裝幾個的老師，因爲舊有殺毒軟件病毒庫過期了一段時間，當我幫他裝上免費殺毒軟件後，一下子殺出三百多條病毒/木馬/風險。
（貌似我最後一次中毒是老媽去打印店帶回來的。）

h3. （三）、同等條件下，開源優於閉源

開源軟件不容易埋藏壞東西，而且一個開發者停止更新了，別人還能繼續。開源是一個趨勢，是人類經過嚴格的版權制度後的一個新走向。開源就像共產，雖然未必現實，畢竟是人類最美好的願景之一，我們何不做做這個美夢呢？開源軟件，讓我們經常不必苦苦等待商家的更新，如果有需要，上網搜一搜，或者自己寫一寫，新功能就出來了。
不過，開源軟件大多難逃坑爹淩亂佔空間的命運， C 語言開發出來的一般還好，現在新的 Ruby, Python 之類，真讓人……

h3. （四）、儘量不用國貨，就算用也儘量用國產外貿貨

原因衹有一個：隱私安全。國貨存在著老大哥的深情凝望問題。安全是最重要的，隱私次之。我寧可把用戶習慣和個人文檔全部透露給米國公司，也不要本國公司知道半點，因爲米國的法律完善，制度合理，又是小政府國家，就算拿走了我的東西，對我也不會有任何負面影響。但中國的我就無法信任，企業道德低下，審查嚴格且警力豐富，我可不想半夜睡著覺，被人敲門查水表。當然，有時候不是你想用外國就用外國的。比如輸入法，我就是想不通爲什麼股溝要把他的輸入法做得這麼爛，爛到我沒法用。還有白癡的阿里旺旺……
另外，中國人由於長期生產力低下，技術落後，所以好東西都做外貿，因此出口轉內銷的東西會好些。這不是崇洋媚外的問題，而是現實。因此，外貿軟件比內銷的又好一些。

h3. （五）、在便攜、綠色、安裝三種體系中正確取捨

軟件的重度使用者一定有兩種苦惱：第一是重裝系統後重裝軟件很麻煩，第二是偶爾想裝某個不常用軟件來用用的時候到處找，找到還要裝半天。這種時候，使用者就會尋求一種不用或不怎麼用安裝的版本。
免安裝的軟件都可以說是綠色軟件，有時也有人把執行一個註冊表就安好的軟件叫綠色軟件。傳統意義上的綠色軟件雖然不必安裝，但重裝或更換電腦後，會丟失他在註冊表或用戶文件夾存儲的信息。比如一款瀏覽器，如果你僅僅使用傳統的綠色版，那麼重裝後，你在瀏覽器保存的帳號、密碼、 cookies 全都會丟失。於是有些人會嫌麻煩，這就產生了便攜版。
便攜版會把所有程序產生的文件都小心的收集起來，重裝後，所有設定都還在，顯得非常方便。有的軟件——通常是一些小工具——是天生的便攜軟件，但大多數工具需要藉助某個系統便攜化。在中國，比較常見的體系有雲端、 ThinApp 、 PortableApps 三大體系。雲端是個準便攜系統，因爲使用還要裝雲端平臺，不過該平臺是個綠色軟件，其特點有三：一是完全傻瓜操作，任何人都能輕易使用安裝包製作雲端版軟件；二是有一個巨大的本土化軟件平臺，包含了所有中國人常用且可便攜化的免費軟件，更新及時，還支持網友製作上傳，因此不論收費免費，都非常容易獲得，有時甚至能找到自己在別處下載不到的軟件（但比較癡線的是，他有很多原生便攜的軟件在裡面）；三是一個很多人不會注意到的特點，那就是他可以很輕易的覺得一款軟件是否一直處於「激活」狀態，這對於一些支持包如 .Net Framework ，或常駐軟件如 Winrar ，再或默認瀏覽器有著很大的優勢。 ThinApp 是個著名的單文件軟件製作工具，也是傻瓜操作，但沒有平臺支持。 PortableApps 是最清潔的便攜軟件，用過之後，軟件會精心把所有痕跡抹掉，還你一個乾淨的系統——聽起來這個體系最有優勢，但他的平臺非常不給力，官方平臺只有幾個小小的免費軟件，非官方平臺非常分散，而且存儲軟件的網盤五花八門，有的非常不好下載。更要命的是 PortableApps 製作便攜軟件非常麻煩，即便軟件熟手如我老人家，也沒有那個耐心一個個研究製作，而往往把軟件製作成「一次性軟件」，而非「便攜軟件」。
根據以上分析，有強迫症的人當然希望自己所有軟件都是 PortableApps 軟件，但其他體系果真沒有生存空間了嗎？也未盡然。
首先，便携软件會把設置程序產生的緩存文件給收集起來，顯然是要加重硬盤負擔的。因此，譬如 UltraISO 、視頻播放器之類基本只需要默認設置的軟件，普通的綠色版就足夠了。
其次， PortableApps 是外國軟件，其中文平臺只有 PortableAppC 一家，站長做幾個主流軟件已經應接不暇了，而中國軟件往往非常流氓，一定要最新版才能使用，這樣的軟件使用雲端平臺就比較有優勢。
同時，雲端這個系統雖然比較「陽春」，但好在軟件多，而且自己製作更方便，因此輸入法、網盤、最常用的那款瀏覽器、一些所有體系都未見綠色版但自己又經常用的軟件，可以使用雲端版。同時，雲端還有一個用途：當沙盒用——有的軟件只想試試效果，那就雲端吧，刪除的時候不費力。
現在我放在雲端平臺上的軟件有十來個，比 PortableApps 略多，但原生便攜和僅需要綠色的軟件更多。
最後，有的軟件根本無法便攜化，或者說難度太高而沒人製作，再或者便攜版、綠色版並不好用，就只好用安裝版了。

h3. （六）、建立好快捷方式

雲端和 PortableApps 都可以在面板看到軟件列表，雲端還支持分類，但兩個面板都比較陽春，雲端的啟動速度更是相當不理想。這樣，每次重裝之後，還是會出現討厭的情況：不記得自己裝了什麼軟件，只好重新看看，再建立快捷方式。
這未免太蠢了。我的方法是：把常用軟件分類放入 Quick Launch ，然後使用 Free Launch Bar ，再把這個文件夾用軟件同步起來。重裝後，打開網盤，裝上 Free Launch Bar ，幾十近百種軟件頓時完美分類呈現，點兩下左鍵就能打開，大大提高使用效率。另外， nrLaunch 也是個不錯的解決方案。
當然，也有人喜歡 Win+R 這種方式。不過重裝頻繁，真的大丈夫嗎？

h3. 戰場之結論

Windows 下軟件很安逸， Linux 下較折騰。如果喜歡嘗試某些開源項目和網上互動， Linux 更好。但娛樂和中國式的辦公、學習則應該在 Windows 下。對於我來說，我最需要的無非就是上網、看電子書、查字、打字、文本編輯，這些 Windows 都做得很好，而 Linux 有很多地方我不大滿意。本來 Windows 不穩定，軟件還不好裝，這個體驗不如 Linux ，但就是因爲體驗不好而用戶極多，現在便攜弄得火紅舒適，從重裝到把軟件全部搞清楚只要一小時，反而很方便重裝，真是世事難料。
從費用的角度來講，我的工具以免費軟件的爲主，但侵權行爲很多：比如用有版權的辭典，還有冒名下載超星書籍，也有一些破解軟件。
