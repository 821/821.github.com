文本編輯器是神馬？可能菜鳥不曾注意這問題，因為在 Windows 和 MAC 下大家都會一般習慣性的裝上一個 Office ，中國的電腦商家和修理工甚至更直接，幫你裝一個帶 Office 的系統完事，很多號稱純淨版的 Ghost Windows 系統都會附上自選安裝 M$ Office 。但是 Office 問題很多，比如性能差，佔用資源多，而且穩定性也實在不怎麼樣。另外雖然 Office 具有排版的功能，卻沒有代碼高亮，不適合進行編程、網頁編輯等等處理，這時候就需要一款專業的文本編輯器來輔助了。
我老人家使用各種文本編輯器的經驗不算豐富，但也不比全世界九成九的人少。我老人家是從不不進行複雜編程，頂多寫寫論文、網頁代碼、小腳本的人，那這篇文章所期待的讀者亦是如此了。文本編輯器、輸入法、鍵盤、鼠標，是數字化時代的筆，找到適合自己的文本編輯器很重要，而且體驗可以說無法複製，每個人都有自己的手感和看法。本文更多的是羅列各款文本編輯器在中文語境下的特色優缺點，並給出自己的選擇。
h2. 起式
很多拳法都有起式，讀者若看過太極拳的視頻，那前面更是要有一段比較長的時間給拳師醞釀。問自己一個問題：我需要什麼？
以下是我的回答：
01. 舒適的學習曲線，權重 3
02. 高性能——我甚少編輯大文件，更看重啟動速度，但秒開和秒替換 5MB 以下文件是需要的，權重 5
03. 代碼及詞彙高亮、代碼折疊等——簡單編程和撰文所需，權重 8
04. 自定義字體、背景顏色等可視效果——方便編輯，權重 8
05. 自動完成詞彙和語法——權重 1
06. 中文支持——是否中文版不重要，關鍵是多種編碼的中文顯示和中英混雜文章的斷行，權重 65535
07. 簡單列模式——能列刪除就好，權重 8
08. 多步撤銷——這個功能都做不出來，還是不要做編輯器了，權重 10
09. 多平臺支持——就是 Windows 和 Linux ，蘋果的系統送我序列號我都不裝，權重 2
10. 替換——正則表達式是基本，我還需要保存替換模式和大批字符串對替換（詳見左文），權重 6
11. 文件比對——權重 2
12. 收藏夾——讓軟件記住所有我的常用文件，權重 6
13. 其他特色功能——我會在每個軟件重點討論，因為多數必要功能是共有的，而特色功能使他們與眾不同
h2. 黑虎掏心 Notepad
哦哦哦，你見過比他還爛的編輯器嗎？？？反正我沒見過。
按理說犧牲了功能的，性能應該要好。可是他偏偏就不行，隨便一篇網上下載的中長篇小說，拿下來想替換幾個字，他就給你裝死……
可是，如果你有虛竹的內力，一招黑虎掏心也能跟鳩摩智打。
好吧，我不是高手，需要高級招式。而且就算是虛竹，不學生死符大概不好收拾丁春秋吧，聽說最近人家還學了降龍十八掌哦。

h2. 乾坤大挪移 Notepad2
Notepad2 和諸般強大的文本編輯器比，顯得非常不給力，但比起 Notepad ，又好了太多，而且還是單文件的。於是我把他下載下來，改名，替換掉 Notepad 。
哦也，從此我電腦上的 Notepad 變成了一個有列模式、可高亮代碼、無限撤銷的高性能編輯器。不過，功能基本上也就這麼多了……
如果需要更多功能，可以選擇其他，或者用他的 <a href="http://code.google.com/p/notepad2-mod/downloads/list" rel="external">MoD</a> 。

h2. 破劍式 Notepad++
Notepad++ 多次被評為最佳免費文字編輯器，運用廣泛。他速度快，界面美觀大方，完全不配置也很好用，是極好的入門編輯器。
Notepad++ 在代碼折疊方面做得很好。不但能折疊，而且對一些語法還有非常漂亮的配對高亮——點擊折疊的一頭就會高亮另一頭。貌似除他我只知道 UltraEdit 原生支持。這對編寫複雜結構的網頁頗為有用，有時 div 多了，改著改著就亂了，如果一點就看到另一頭，就能比較輕鬆的看到另一頭的位置，縮排規範的話，立刻會發現問題。
我也非常喜歡他的選中高亮，很多軟件選中一個詞之後，只有被選中的詞高亮起來，而 Notepad++ 是高亮所有同一詞彙。
問題來了：編輯中英混雜的文章時，會把連續沒空格的中文視為一個單詞，所以自動換行非常囧。諷刺的是，作者是台灣人。我真的想問一問：你自己到底用沒用過 Notepad++ ？還有就是他的搜索有問題，比如 html ，搜索 htm 就找不到，這樣中文搜索也是有問題的。於是， Notepad++ 只能作為我的代碼編輯器了。他的設置也比較複雜，甚至可以說相當坑爹。所以當你需要進階功能時，就會放棄他。

h2. 降龍十八掌 UltraEdit
曾幾何時， UltraEdit 在業界地位極高，大有圖形化設置的編輯器中老大的地位，基本上新版一出，盜版跟進。
原因有幾點：出道早，易上手，支持十六進制編輯，強大的列模式，自定義配色，當時還真沒幾樣編輯器能做到。
悲劇的是，長串中文在 UltraEdit 還是一個詞。順便說一下，默認工具欄很爛，保存都沒有，就一堆不知所謂的爛圖標，不知作者在想什麼……現在 UltraEdit 最大的優勢也就是重要功能他都有，外加一個極品列模式了。

h2. 唐詩劍法 EmEditor
泥轟人作品，可能是亞洲語系長期被邊緣的緣故吧，作者對各種語系都做得相當週到。總體來說，這軟件非常細膩，視覺上很舒適。
不過如果不設置一下，又不肯認真學習技術，那真不行，整一個戚芳版躺屍劍法。可是設置過後就變成唐詩劍法啦。但他真正吸引我的，是設置即可標點符號上色，和一些精緻的細節。比如保存按鈕，如果已經保存了，就會顯示為灰色，直到有了修改，才是彩色。看到灰色的保存鍵，我就知道不必保存了。一些新興軟件，比如 AkelPad ，就吸收了。他的大綱功能也很給力，可以通過自定義設置，把需要凸顯的章節露出來，只要點右側的大綱欄，就能快速跳到你設置的點。這不同於書籤功能。書籤是你把光標移動到某處，然後設置。大綱則是在配置裡設定好關鍵字符組合，以後每次出現該字符組合，不用再設置，自動大綱化。
EmEditor 的替換也代表着當代單文件替換技術的最高水準。可能有的人會聽過 Search & Replace 和 Actual Search & Replace 這對機油，號稱批量查找替換界的倚天劍和屠龍刀。不過我把他們都刪掉了，因爲我很少有機會去替換大批文本文件，但我經常替換一個文本文件的多對字符串。一個合格的文本編輯器應該具備正則表達式替換，而一個優秀的文本編輯器似乎就不應滿足於正則表達式，而應該引入批量字符串對的替換了。比如說，我想把全拼碼表換成雙拼，那就有三四十對字符串要替換，一對對輸入、替換的話當然可以，但實在太麻煩了。而且我可能有很多個碼表要做替換工作，那麼保存替換腳本也是很重要的。也許有人會舉起小手提到馬健寫的 textforever ，不過那玩意的替換跟 Search & Replace 之類差不多，還不支持非系統默認編碼的文件，而且替換腳本的格式不懂是十六進制還是神馬，反正很難編輯，只能一對一對添加。 EmEditor 就有字符串批量替換功能，而且替換腳本是 tsv 格式，非常方便修改。當然了，宏也是各家文本編輯器實現這個功能的手段之一，可是到底不如原生支持來得方便，還要自己寫宏。
代碼片段也是 EmEditor 的特色，有了此物，你甚至覺得 Zen Coding 的 html 簡寫工具都是多餘的，因為論可定制性，還是代碼片段來得美妙。但我不知道為什麼作者把這功能做成插件，讓我每次想用都要啟動插件，而且啟動插件會出現一個側邊欄……這一點就不如 Gedit/Kate 簡單了。
EmEditor 功能上最大的缺陷是大概就是沒有收藏夾罷，收藏夾可以用項目插件來代替，不過還是真正的收藏夾更好用。另外，做大型項目的程序猿也會說他不適合打造成 IDE 。總之， EmEditor 是一款近乎完美的大眾化文本編輯器，在我看來，他是所有文本編輯器中最積極吸收他者優點的一款。

h2. 神行百變 EditPlus
EditPlus 下載回來，我一開，哇靠，山寨版 EmEditor （作者是棒子，果然是山寨 EmEditor ）。
EditPlus 小巧輕便性能高，界面和功能與 EmEditor 頗為相似，卻不像 EmEditor 那樣需要較多設置才能用，門檻低，速度快，韋小寶都能用。他的 html 工具條很別家的頗為不同，代碼自動完成也做得不錯，還有個「素材」功能，感覺有點像 cheatsheet 。
按說 EditPlus 的個性並不強，但他有一個其他軟件沒見過的功能——窗口平鋪。比如你開了十個文件，只要輕輕點一下平鋪，他立刻把這十個窗口根據一個比較合理的配置全部呈現在用戶眼前，有點 MAC 的感覺是不是，還可以自定義每個窗口的大小。所以我把 EditPlus 當成筆記管理軟件。功能夠，速度快，做出來的筆記毫無軟件依賴性。而且由於 EditPlus 內建了一個 IE ，不用專門開瀏覽器就可以預覽 html ，同時支持用指定瀏覽器打開當前文檔，這都是前端開發非常好用的功能。快捷鍵也是他的特色，你可以設置菜單欄裡所有命令的快捷鍵，只要你記得。如果寫有函數的腳本，他也支持函數定位，不錯的功能——他和 UE 一樣，都屬於功能過剩的，主要看你需要不需要和發現沒發現。
不過他的可定制性稍差（可定制性和設置難度從來就是一對矛盾，然而我覺得 EditPlus 的平衡點不錯），一些方面不如 EmEditor 細膩。代碼高亮也比較原始，嗯，缺點內功……

h2. 降龍十八缺三掌 PsPad
被譽為 UltraEdit 的最佳免費替代品，到底有沒有這麼厲害呢？
我嘗試之後，覺得他跟 UltraEdit 的差距還是比較大的，很多功能都沒做好，甚至自動換行都曾經被作者刪掉，簡單的代碼折疊居然都沒有。而且不是 unicode 軟件。結果 PsPad 就像缺了幾招的降龍十八掌一樣，怎麼看也就是黎生的水準（知道黎生是誰嗎，補習一下射雕吧）。

h2. 北冥神功 AkelPad
新興的免費瀏覽器，插件多，速度快。用這軟件嚴重推薦烈火漢化版，總是加上很多插件和按鈕（烈火作品吸引我的從來不是漢化本身，而是破解和加強）。如果作者有心把插件整合一下，再把設置功能做友好些，完全可以賣個好價。剛接觸 AkelPad 、第一次寫本文的時候，他還顯得頗簡陋，祇覺得他很有發展前景，關注了一下。但他本身實在進步神速，而且烈火似乎也相當積極把插件加進去，因此時隔一年再用烈火版，他忽然變成 GUI 編輯神器了。
他很早就實現了拆分窗格功能。當我們編輯長代碼或長篇文章的時候，如果想同時瞻前又顧後，拆分一下窗口就可以同時了，最多可以拆分成四個窗口，絕對夠用了。另外他支持最小化到任務列，這是一般文本編輯器所不具備的。
不過最吸引我的還是收藏夾。我有一些經常編輯的軟件，總數還不小，資源管理器又祇管一個文件夾，這實在讓人苦惱。有了收藏夾，問題就解決了。
AkelPad 選項比 EditPlus 還少，但有些更像是不同編輯器的取捨的東西，他也允許設置。比如，你可以設置查找替換是否循環，再有空格符、製表符這些，雖然軟件有內建，都是可以通過插件修改的。最神奇的是他的折行方式選擇，選按符號，就是我比較滿意的中文折行，選按單詞，就是類似 UE 等中文支持較差的軟件的方式。開發者沒能像 EmEditor 和 EditPlus 那樣完美解決中英混雜文章的折行問題，卻又提供了另一思路。
不過，目前 AkelPad 還有一些不足之處。比如他的列模式就不如 UltraEdit, EmEditor 強大。代碼高亮的可定制性也弱於成熟軟件。他的代碼編輯體驗還算及格，我比較喜歡他的代碼大綱，用起來還挺像 EmEditor 的大綱，也提供了輸出為富文本的功能。

h2. 葵花寶典 VIM
VIM 功能強大，又不是那麼好學，而且門外漢往往敲了半天都輸不進一個字。反正練對了就東方不敗，沒練對就林震南。不過平心而論，我覺得 VIM 不是一個速度特別好的編輯器，特別是編輯大文件的時候，體驗明顯不如 EmEditor （當然這廝的實現手段很偷懶），因而編輯大文件時，還是不能依賴他。
VIM 的特點在於命令操作，使用 VIM 可說是一種鍵盤操作體驗——前提是你記得命令。由於 VIM 把操作分為三種模式，所以不太用組合鍵，這樣手指的壓力會減小很多。我用得最熟的命令就是 @dd@ （比 UltraEdit 下的 @Ctrl+E@ 好按，更不必說 EditPlus 的 @Alt+Shift+Delete@ 了，誰想出這麼二逼文藝青年的組合鍵啊，而且 @dd@ 可以用來多行刪除）。還有比如想截取一個比較大的文件的若干行，那必然是 VIM 最給力，速度快不說，幾個命令也不難記。但是比如替換之類的，我就覺得非常不給力。再有模式切換也有討厭的地方，比如很多重要功能需要冒號進入模式，這一點非常不好，因為冒號的輸入需要按著 @Shift@ ……
有鑒於 VIM 的命令比較好用，很多現代化文本編輯器都會集成 VIM 模式。有的做個插件，有的原生支持。所以其實不用他也不必羨慕。好按的快捷鍵已經不是 VIM 獨有。
VIM 高手一般會把工具欄都殺光，欲練神功，黃且暴力，所以如果還想跟鼠標洞房花燭，那就衹能找楊蓮亭了 <img src='{{ site.url }}/assets/Smilies/o042.gif'> 。

h2. 太玄經 Emacs
Emacs 是偽裝成編輯器的操作系統，因為如果你會設置，他簡直無所不能。
我注意到 Emacs 是因為 LaTeX 。寫長文章修改的時候比較怕遇到代碼密集區，那真是不堪入目，所以最好能有一款一邊寫一邊 preview 的。我看了一下，基本上都是收費還找不到破解版的，或者就是問題比較多的，唯獨 Emacs 配合一些插件貌似沒人非議，所以很想試試看。裝上之後，我就知道不可能了。 VIM 已經夠難，沒想到 Emacs 更難，上來就要你進行一大堆的設置。你就不能發一個編輯好參數文件的版本麼，高手愛怎麼設置是他的事，可是你總該給一個低手能用的吧。另外，同是鍵盤操作， Emacs 不如 VIM 好用，除非裝上腳踏板，特別對我是我這種手指很短的人，最討厭按 @Ctrl@ 了。如果說 VIM 是二逼青年的編輯器，那 Emacs 絕對是二逼文藝青年的編輯器。

h2. 岱宗如何 SciTE
跨平臺的強大編輯器，說起來他和 Notepad++ 及 Notepad2 還是兄弟，而且 SciTE 是嫡子——他們的老爸叫 Scintilla 。但和 Notepad++ 定位不同，他注重定制，需要到參數文件自行配置，以及用 lua 進行功能擴展。但 SciTE 的配置文件列出了幾乎所有可定制內容，並將沒用到的都注釋起來，所以用戶可以非常輕易的在這模版下摸索，而 VIM 和 Emacs 都沒有這麼做（也不好做）。故此， SciTE 的地位非常尷尬，功能不到極致，友好度又不及其他，所以強大是強大，小巧是小巧，免費是免費，卻無法取得統治地位。
由於開源且 lua 插件比較容易做，在很多語言及腳本圈子內部， SciTE 是很受歡迎的，還有人提供配置好了的下載，可見他對代碼編輯的優勢。但這些修改往往不觸及內核，以致他的修改版都顯得雷同且有些僵化。插件作者似乎比較熱衷於對自己常用的某款語言進行優化，而一般性的插件則甚少開發，導致目前功能性質的插件還比較少。

h2. 半部葵花寶典+楊蓮亭 Editra
Editra 原生支持 VI 模式，可以在一個圖形化的窗口中直接選擇不同語言下的顏色方案，並且實時預覽顏色效果。他還是三大系統通吃的，這種編輯器往往直接編輯配置文件，而他堅持 GUI 。
遺憾的是，中英混雜文件的折行問題又出現了……而且似乎對 ansi 的中文顯示有比較多問題。

h2. 沖靈劍法 Sublime Text 2
忽然火起來的一個編輯器，介紹說，他介面是仿蘋果下 TextMate 的，原生支持 VI 模式，可以快速定位函數，多行編輯，還能右鍵打開鏈接裡的本地文件。於是我下載來看了看，結果發現槽點實在太多了，如果說別的編輯器的缺點叫瑕不掩瑜， ST2 的缺點叫千瘡百孔。
首先他的中文支持是最差的，不但中文折行很渣，而且不能顯示 GBK ，祇能用插件轉換成 unicode 再用。單是這理由，我就把他槍斃了，我天天都寫中文，怎能不完美支持中文？不過我還是可以繼續批判。這貨右側有個文件的縮影，截圖的時候看起來很牛，實際一用覺得完全垃圾。根本看不清，還好可以關掉。
我找了半天也沒發現工具欄，最後百度發現需要用 @Ctrl+Shift+P@ （沒錯， Emacs 的快捷鍵跟他比起來簡直弱爆了）臨時呼叫出一豎條，還一眼望不到底的要下拉那種，權當工具欄。這是我見過最醜、最難用的工具欄。
很多人說他的配色方案很好，屁。他的 GUI 只能在幾套方案裡選那幾種別的編輯器都有的，要自定義還要改代碼。這算是哪門子好？
還有人談到啟動速度。這個沒錯，但是開大文件不快。其實我要開的文件也不大，一般都五兆以內，但我測試覺得他明顯比別人慢了。再說文本編輯器哪有打開特慢的？又不是辦公軟件。
我沒實際使用，祇進行了常用功能的測試，但據說此軟件 bug 並不少。就這半成品，也敢叫 59 刀的天價，實在匪夷所思。如果編輯純英文的文檔，他還有點用；如果拿來編輯了中文文檔還說他好，我建議你到街上裸奔，並說你穿了最新潮的衣服。

h2. 乾坤大挪移 jEdit
VIM 和 Emacs 的粉絲將他們捧上天，但如果這麼好，再加上二者免費、跨平臺、還這麼老，早就千秋萬載一統江湖了。不過東方不敗沒做到，無名太監也沒做到，君子劍先生甚至死於小菜鳥儀琳之手。很多人把不用 VIM 和 Emacs 歸結為太懶，這祇能說明他們在拒絕理解他們無法理解的事物。人們發明鼠標，就是因為單用鍵盤不夠方便。很多事可以單手鼠標完成，何必雙手？你以為人人都住在熱帶，不會遇到冬天不想把手伸出被子的情況嗎？鼠標還能拿到離自己最近的地方，然後把腳搭在舒適的地方（如桌子上），鍵盤流恐怕很難罷？
jEdit 就是一個強大但同時滿足鼠標流的現代化跨平臺編輯器。雖然用戶還較少，插件不多，軟件已經有很多方面做得不錯。他的插件管理非常方便，插件管理器裡勾選下載後，不必重啟軟件本身，而且插件的文檔、設置都跟編輯器本身的文檔、設置整合在一起。另外我也喜歡他的收藏夾，這好像是唯一一個自帶收藏夾的編輯器？還有很多特色功能都可以在他這裡找到，比如原生 VIM 模式之類。
但缺點也很多。他需要 Java 環境，啟動慢（但使用中切換沒問題，這一點讓人比較開心）。由於亞洲用戶還不多，中文字體渲染也有點問題，有的渲染會導致部分中文字符閃爍，且這些渲染會導致使用中文輸入法編輯時，自動折行出現一些問題。還有 Windows 下中文輸入法的顯示也不對勁，這會影響作者的打字效率——除非使用重碼率極低、可以盲打的形碼。另外，他有些設置很奇怪，比如行末標記是個很小的點，太不明顯，而且位置跟英文句號很接近，容易搞錯。空格和製表符並不原生可見，插件作者審美有點毛病，用非常不舒服的一個小圈表示空格。還有他不適合編輯過大的文件，開大文件還要設置半天。
強大是強大，可這苦逼的娃就像乾坤大挪移，有幾句話你就是張無忌也練不成。

h2. 收式
文本編輯器和輸入法都是非常私人的東西，沒有絕對的好，但似乎有絕對的差。對我而言， EmEditor 是最舒適的一款， AkelPad 和 EditPlus 也不錯。如果使用 Linux ，我會比較願意使用 SciTE 和 jEdit ，但二者都不是我非常滿意的編輯器。目前暫時找不到一款讓我完全滿意並停止尋找之路的編輯器。
