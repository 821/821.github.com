文本編輯器是神馬？可能菜鳥不曾注意這問題，因爲在 Windows 和 MAC 下大家都會一般習慣性的裝上一個 Office ，中國的電腦商家和修理工甚至更直接，幫你裝一個帶 Office 的系統完事，很多號稱純淨版的 Ghost 系統都會附上自選安裝 M$ Office 。但是 Office 問題很多，比如性能差，佔用資源多，而且穩定性也實在不怎麼樣。另外雖然 Office 具有排版的功能，卻沒有代碼高亮，不適合進行編程、網頁編輯等等處理，這時候就需要一款專業的文本編輯器來輔助了。
我老人家使用各種文本編輯器的經驗不算豐富，但也不比全世界九成九的人少。我老人家從不進行複雜編程，頂多寫寫論文、網頁代碼、小腳本的人，那這篇文章所期待的讀者亦是如此了。文本編輯器、輸入法、鍵盤、鼠標，是數字化時代的筆，找到適合自己的文本編輯器很重要，而且體驗可以說無法複製，每個人都有自己的手感和看法。本文更多的是羅列各款文本編輯器在中文語境下的特色優缺點，並給出自己的選擇。

h2. 起式

很多拳法都有起式，讀者若看過太極拳的視頻，那前面更是要有一段比較長的時間給拳師醞釀。問自己一個問題：我需要什麼？
以下是我的回答：
01. 舒適的學習曲線，權重 6
02. 高性能——啟動速度快，秒開和秒替換 5MB 以下文件，權重 6
03. 代碼及詞彙高亮、代碼折疊等——簡單編程和撰文所需，權重 8
04. 自定義字體、背景顏色等可視效果——需要方便編輯，權重 10
05. 自動完成、代碼縮寫——看視頻很美好，其實不常用，權重 1
06. 中文支持——是否中文版不重要，關鍵是多種編碼的中文顯示和中英混雜文章的斷行，權重 65535
07. 簡單列模式——能列刪除就好，權重 5
08. 多步撤銷——這個功能都做不出來，還是不要做編輯器了，權重 10
09. 多平臺支持——就是 Windows 和 Linux ，權重 2
10. 替換——正則表達式是基本，還需要保存替換模式和大批字符串對替換（詳後），權重 8
11. 文件比對——權重 1
12. 收藏夾——讓軟件記住所有常用文件，權重 6
13. 自動保存——雖然我有手動保存強迫症，不等自動保存就會手動保存掉，但既然有強迫症，權重 8 好了
14. 方便添加特殊功能到工具欄——經常寫點 bat ，然後做成圖標，放到編輯器裏用，權重 7
15. 自定義快捷鍵——不會定義很多，多了記不住的，權重 1
16. 宏——偶爾錄製一下，但是頻率很低，權重 1
17. 其他特色功能

h2. 黑虎掏心 Notepad

哦哦哦，你見過比他還爛的編輯器嗎？？？反正我沒見過。
按理說犧牲了功能的，性能應該要好。可是他偏偏就不行，隨便一篇網上下載的中長篇小說，拿下來想替換幾個字，他就給你裝死……
可是，如果你有虛竹的內力，一招黑虎掏心也能跟鳩摩智打。
好吧，我不是高手，需要高級招式。而且就算是虛竹，不學生死符大概不好收拾丁春秋吧，聽說最近人家還學了降龍十八掌哦。

h2. 乾坤大挪移 Notepad2

Notepad2 和諸般強大的文本編輯器比，顯得非常不給力，但比起 Notepad ，又好了太多，而且還是單文件的。於是我把他下載下來，改名，替換掉 Notepad 。
哦也，從此電腦上的 Notepad 變成了一個有列模式、可高亮代碼、無限撤銷的高性能編輯器。不過，功能基本上也就這麼多了……
如果需要更多功能，可以選擇其他，或者用他的 <a href="http://code.google.com/p/notepad2-mod/downloads/list" rel="external">MoD</a> 。

h2. 破劍式 Notepad++

Notepad++ 多次被評爲最佳免費文字編輯器，運用廣泛。他速度快，界面美觀，完全不配置也很好用，是極好的入門編輯器。
Notepad++ 在代碼折疊方面做得很好。不但能折疊，而且對一些語法還有非常漂亮的配對高亮。這對編寫複雜結構的網頁頗爲有用，有時 div 多了，改改就亂了，如果一點就看到另一頭，就能比較輕鬆的看到另一頭的位置，縮排規範的話，立刻會發現問題。
我也非常喜歡他的選中高亮，很多軟件選中一個詞之後，只有被選中的詞高亮起來，而 Notepad++ 是高亮所有同一詞彙。
問題來了：編輯中英混雜的文章時，會把連續沒空格的中文視爲一個單詞，自動換行非常囧。諷刺的是，作者是台灣人。我真的想問一問：你自己到底用沒用過 Notepad++ ？還有他的搜索有問題，比如 html ，搜索 htm 就找不到，這樣中文搜索也是有問題的。看來 Notepad++ 只能作爲代碼編輯器了。
他的設置也比較複雜，甚至可以說相當坑爹。所以當你需要進階功能時，就會放棄他。

h2. 降龍十八掌 UltraEdit

曾幾何時， UltraEdit 在業界地位極高，大有圖形化設置的編輯器中老大的地位，基本上新版一出，盜版跟進。
原因有幾點：出道早，易上手，支持十六進制，列模式，自定義配色，當時還真沒幾樣編輯器能做到。
悲劇的是，長串中文在 UltraEdit 還是一個詞。他的默認工具欄很爛，保存都沒有，就一堆不知所謂的爛圖標，不知作者在想什麼……他的 html 工具欄還行，不過我用的機會不多。現在 UltraEdit 最大的優勢也就是重要功能他都有，外加一個極品列模式了。

h2. 唐詩劍法 EmEditor

泥轟人作品，可能是亞洲語系長期被邊緣的緣故吧，作者對各種語系都做得相當週到。總體來說，這軟件非常細膩舒適。
如果不設置一下，那真不行，整一個戚芳版躺屍劍法，設置過後就變成唐詩劍法啦。他真正吸引我的，是一些精緻的細節。比如標點符號，一般軟件並不會高亮，但 EmEditor 可以輕易設置成高亮的，甚至把中英字符弄成不同顏色。他的大綱功能也很給力，可以通過自定義設置，把需要凸顯的章節露出來，只要點右側的大綱欄，就能快速跳到你設置的點。這不同於書籤功能。書籤是你把光標移動到某處，然後設置。大綱 (Outline) 則是在配置裡設定好關鍵字符組合，以後每次出現該字符組合，不用再設置，自動大綱化。
EmEditor 的替換也代表着當代單文件替換技術的最高水準。可能有的人會聽過 Search & Replace 和 Actual Search & Replace 這對機油，號稱批量查找替換界的倚天劍和屠龍刀。不過我把他們都刪掉了，因爲我很少有機會去替換大批文本文件，但我經常替換一個文本文件的多對字符串。一個合格的文本編輯器應該具備正則表達式替換，而一個優秀的文本編輯器似乎就不應滿足於正則表達式，而應該引入批量字符串對的替換了。比如，我想把全拼碼表換成雙拼，那就有三四十對字符串要替換，一對對輸入、替換當然可以，但實在太麻煩了。而且我可能有很多個碼表要做替換工作，那麼保存替換腳本也是很重要的。也許有人會舉起小手提到馬健寫的 textforever ，不過那玩意的替換跟 Search & Replace 之類差不多，還不支持非系統默認編碼的文件，而且替換腳本的格式不懂是十六進制還是神馬，反正很難編輯，只能一對一對添加。 EmEditor 就有字符串批量替換功能，而且替換腳本是 tsv 格式，非常方便修改。當然了，宏也是各家文本編輯器實現這個功能的手段之一，可是到底不如原生支持來得方便，還要自己寫宏。
代碼片段 (Snippets) 也是 EmEditor 的特色，有了此物，你甚至覺得 Zen Coding 的 html 簡寫工具都是多餘的，因爲論可定制性，還是代碼片段來得美妙。但我不知道爲什麼作者把這功能做成插件，讓我每次想用都要啟動插件，而且啟動插件會出現一個側邊欄……這一點就不如 Gedit/Kate 簡單了。
EmEditor 功能上最大的缺陷是大概就是沒有收藏夾罷，收藏夾可以用項目 (Project) 插件來代替，不過還是真正的收藏夾更好用。另外，做大型項目的程序猿也會說他不適合打造成 IDE 。
EmEditor 是一款近乎完美的大眾化文本編輯器，是所有文本編輯器中最積極吸收他者優點的一款。

h2. 神行百變 EditPlus

EditPlus 下載回來，我一開，哇靠，山寨版 EmEditor （作者是棒子，果然是山寨 EmEditor ）。
EditPlus 小巧輕便性能高，界面和功能與 EmEditor 頗爲相似，卻不像 EmEditor 那樣需要較多設置才能用，門檻低，速度快，韋小寶都能用。他的 html 工具條很別家的頗爲不同，代碼自動完成也做得不錯，還有個「素材」功能，感覺有點像 cheatsheet 。
按說 EditPlus 的個性並不強，但他有一個其他軟件沒見過的功能——窗口平鋪。比如你開了十個文件，只要輕輕點一下平鋪，他立刻把這十個窗口根據一個比較合理的配置全部呈現在用戶眼前，有點 MAC 的感覺是不是，還可以自定義每個窗口的大小。快捷鍵也是他的特色，你可以設置菜單欄裡所有命令的快捷鍵，只要你記得。如果寫有函數的腳本，他也支持函數定位，不錯的功能——他和 UE 一樣，都屬於功能過剩的，主要看你需要不需要和發現沒發現。
不過他的可定制性稍差（可定制性和設置難度從來就是一對矛盾，然而我覺得 EditPlus 的平衡點不錯），一些方面不如 EmEditor 細膩。代碼高亮也比較原始，嗯，缺點內功……

h2. 降龍十八缺三掌 PsPad

被譽爲 UltraEdit 的最佳免費替代品，到底有沒有這麼厲害呢？
我嘗試之後，覺得他跟 UltraEdit 的差距還是比較大的，很多功能都沒做好，甚至自動換行都曾經被作者刪掉，簡單的代碼折疊居然都沒有。而且不是 unicode 軟件。結果 PsPad 就像缺了幾招的降龍十八掌一樣，怎麼看也就是黎生的水準（知道黎生是誰嗎，補習一下射雕吧）。

h2. 北冥神功 AkelPad

新興的免費瀏覽器，插件多，速度快。用這軟件嚴重推薦烈火漢化版，總是加上很多插件（烈火作品吸引我的從來不是漢化本身，而是破解和加強）。如果作者有心把插件整合一下，再把設置和功能調用做友好些，完全可以賣個好價。剛接觸 AkelPad 、第一次寫本文的時候，他還顯得頗簡陋，衹覺得他很有發展前景，關注了一下。但他本身實在進步神速，而且烈火似乎也相當積極把插件加進去，因此時隔一年再用烈火版，他忽然變成 GUI 編輯神器了。
他很早就實現了拆分窗格功能。當我們編輯長代碼或長篇文章的時候，如果想同時瞻前又顧後，拆分一下窗口就可以同時了，最多可以拆分成四個窗口，絕對夠用了。另外他支持最小化到任務列，這是一般文本編輯器所不具備的。
不過最吸引我的還是收藏夾。我有一些經常編輯的軟件，總數還不小，資源管理器又衹管一個文件夾，這實在讓人苦惱。有了收藏夾，問題就解決了。
AkelPad 選項比 EditPlus 還少，但有些更像是不同編輯器的取捨的東西，他也允許設置。比如，你可以設置查找替換是否循環，再有空格符、製表符這些，雖然軟件有內建，都是可以通過插件修改的。最神奇的是他的折行方式選擇，選按符號，就是我比較滿意的中文折行，選按單詞，就是類似 UE 等中文支持較差的軟件的方式。開發者沒能像 EmEditor 和 EditPlus 那樣完美解決中英混雜文章的折行問題，卻又提供了另一思路。
不過，目前 AkelPad 還有一些不足之處。比如他的列模式就不如 UltraEdit, EmEditor 強大。代碼高亮的可定制性也弱於成熟軟件。他的代碼編輯體驗還算及格，我比較喜歡他的代碼大綱，用起來還挺像 EmEditor 的大綱，也提供了輸出爲富文本的功能。

h2. 葵花寶典 VIM/gVIM

VIM 強大，又不是那麼好學，門外漢往往敲了半天都輸不進一個字。練對了就東方不敗，沒練對就林震南。不過平心而論，我覺得 Windows 下的 gVIM 不是很快，特別是編輯大文件時，體驗明顯不如 EmEditor （當然這廝的實現手段很偷懶）。
VIM 的特點在於命令操作，使用 VIM 可說是一種鍵盤操作體驗——前提是記得命令。由於 VIM 把操作分爲三種模式，所以不太用組合鍵，手指的壓力會減小很多。我用得最熟的命令就是 @dd@ （比 UltraEdit 下的 @Ctrl+E@ 好按，更不必說 EditPlus 的 @Alt+Shift+Delete@ 了，誰想出這麼二逼文藝的組合鍵啊，而且 @dd@ 可以用來多行刪除，還好這些都可以自定義）。還有比如想截取一個比較大的文件的若干行，那必是 VIM 最給力，速度快不說，幾個命令也不難記。但是比如替換之類的，我就覺得非常不給力。再有模式切換也有討厭的地方，很多重要功能需要冒號進入模式，這一點非常不好，因爲冒號的輸入需要按著 @Shift@ ……
有鑒於 VIM 的命令較好用，很多現代化文本編輯器都會集成 VIM 模式。有的做個插件，有的原生支持。所以其實不用他也不必羨慕。好按的快捷鍵已不是 VIM 獨有。
VIM 高手一般會把工具欄都殺光，欲練神功，黃且暴力，所以如果還想跟鼠標洞房花燭，那就衹能找楊蓮亭了 <img src='{{ site.url }}/assets/Smilies/o042.gif'> 。

h2. 太玄經 Emacs

偽裝成編輯器的操作系統，如果你會設置，他簡直無所不能。
我注意到 Emacs 是因爲 TeX 。寫長文章修改的時候比較怕遇到代碼密集區，簡直不堪入目，所以最好能有一款一邊寫一邊 preview 的。我看了一下，基本上都是收費還找不到破解版的，或者就是問題比較多的，唯獨 Emacs 配合一些插件貌似沒人非議，所以很想試試看。裝上之後，我就知道不可能了。 VIM 已經夠難，沒想到 Emacs 更難，上來就要你進行一大堆的設置。你就不能發一個編輯好參數文件的版本麼，高手愛怎麼設置是他的事，可是你總該給一個低手能用的吧。另外，同是鍵盤操作， Emacs 不如 VIM 好用，除非裝上腳踏板，特別對我是我這種手指很短的人，最討厭按 @Ctrl@ 了。如果說 VIM 是二逼青年的編輯器，那 Emacs 絕對是二逼文藝青年的編輯器。
當然了，如果讓我免學習代價完全掌握一款文本編輯器，我當然會選他。問題是他的時間成本眞的太高了。學會了 VIM 或者 Emacs 的人一般不會換成別的編輯器，原因是這兩款的學習成本太高，學會了還換那也太不拿時間當生命了。不過 gVIM 和 Emacs 在 Windows 下的體驗眞心不好， Linux 下的 VIM 那種迅捷美妙完全不見了，而 Emacs 比較擅長和 *nix 系統本身進行深度整合，在 Windows 下處於被閹割狀態，所以我本人不推薦在 Windows 下使用這兩款軟件。

h2. 岱宗如何 SciTE

跨平臺的強大編輯器，說起來他和 Notepad++ 及 Notepad2 還是兄弟，而且 SciTE 是嫡子——他們的老爸叫 Scintilla 。但和 Notepad++ 定位不同，他注重定制，需要到參數文件自行配置，以及用 lua 進行功能擴展。但 SciTE 的配置文件列出了幾乎所有可定制內容，並將沒用到的都注釋起來，所以用戶可以非常輕易的在這模版下摸索，而 VIM 和 Emacs 都沒有這麼做（也不好做）。故此， SciTE 的地位非常尷尬，功能不到極致，友好度又不及其他，所以強大是強大，小巧是小巧，免費是免費，卻無法取得統治地位。
由於開源且 lua 插件比較容易做，在很多語言及腳本圈子內部， SciTE 是很受歡迎的，還有人提供配置好了的下載，可見他對代碼編輯的優勢。但這些修改往往不觸及內核，以致他的修改版都顯得雷同且有些僵化。插件作者似乎比較熱衷於對自己常用的某款語言進行優化，而一般性的插件則甚少開發，導致目前功能性質的插件還比較少。

h2. 半部葵花寶典+楊蓮亭 Editra

Editra 原生支持 VI 模式，可以在一個圖形化的窗口中直接選擇不同語言下的顏色方案，並且實時預覽顏色效果。他還是三大系統通吃的，這種編輯器往往直接編輯配置文件，而他堅持 GUI 。
遺憾的是，中英混雜文件的折行問題又出現了……而且似乎對中文顯示有比較多問題。

h2. 沖靈劍法 Sublime Text

忽然火起來的一個編輯器，介紹說，他介面是仿蘋果下 TextMate 的，原生支持 VI 模式，可快速定位函數，多行編輯，還能右鍵打開鏈接裡的本地文件。於是我下載來看了看，結果發現槽點實在太多了，如果說別的編輯器的缺點叫瑕不掩瑜， ST 的缺點叫千瘡百孔。
首先他的中文支持是最差的，不但中文折行很渣，而且不能顯示 GBK ，衹能用插件轉成 unicode 再用。單是這理由，我就把他槍斃了，我天天都寫中文，怎能不完美支持中文？不過我還是可以繼續批判。這貨右側有個文件的縮影，截圖的時候看起來很牛，實際一用完全垃圾。根本看不清，還好可以關掉。
我找了半天也沒發現工具欄，最後百度發現需要用 @Ctrl+Shift+P@ （沒錯， Emacs 的快捷鍵跟他比起來簡直弱爆了）臨時呼叫出一豎條，還一眼望不到底的要下拉那種，權當工具欄。這是我見過最醜、最難用的工具欄。
很多人說他的配色方案很好，屁。他的 GUI 只能在幾套方案裡選那幾種別的編輯器都有的，要自定義還要改代碼。這算哪門子好？
還有人談到啟動速度。這個沒錯，但是開大文件不快。其實我要開的文件也不大，一般都五兆以內，但我測試覺得他明顯比別人慢了。再說文本編輯器哪有打開特慢的？又不是辦公軟件。
我沒實際使用，衹進行了常用功能的測試，覺得功能欠缺嚴重，免費的都懶得用，何況還要破解。如果編輯純英文的文檔，他還有點用；如果拿來編輯了中文文檔還說他好，我建議你到街上裸奔，並說你穿了最新潮的衣服。

h2. 乾坤大挪移 jEdit

VIM 和 Emacs 的粉絲將他們捧上天，但如果這麼好，再加上二者免費、跨平臺、還這麼老，早就千秋萬載一統江湖了。不過東方不敗沒做到，無名太監也沒做到，君子劍先生甚至死於小菜鳥儀琳之手。很多人把不用 VIM 和 Emacs 歸結爲太懶，這衹能說明他們在拒絕理解他們無法理解的事物。人們發明鼠標，就是因爲單用鍵盤不夠方便。很多事可以單手鼠標完成，何必雙手？你以爲人人都住在熱帶，不會遇到冬天不想把手伸出被子的情況嗎？鼠標還能拿到離自己最近的地方，然後把腳搭在舒適的地方（如桌子上），鍵盤流恐怕很難罷？
jEdit 就是一個強大但同時滿足鼠標流的現代化跨平臺編輯器。雖然用戶還較少，插件不多，軟件已經有很多方面做得不錯。他的插件管理非常方便，插件管理器裡勾選下載後，不必重啟軟件，而且插件的文檔、設置都跟編輯器本身的文檔、設置整合在一起。另外我也喜歡他的收藏夾，這好像是唯一一個自帶收藏夾的編輯器？還有很多特色功能都可以在他這裡找到，比如原生 VIM 模式之類。
但缺點也很多。他需要 Java 環境，冷啟動慢（但熱啟動沒問題）。由於亞洲用戶還不多，中文字體渲染也有點問題，有的渲染會導致部分中文字符閃爍，且這些渲染會導致使用中文輸入法編輯時，自動折行出現一些問題，用了渲染後鼠標滾輪滾動瀏覽不流暢。輸入法顯示也不對勁，這會影響作者的打字效率——除非使用重碼率極低、可以盲打的形碼。另外，他有些設置很奇怪，比如行末標記是個很小的點，太不明顯，而且位置跟英文句號很接近，容易搞錯。空格和製表符並不原生可見，插件作者審美有點毛病，用非常不舒服的一個小圈表示空格。還有他不適合編輯過大的文件，開大文件還要設置半天。
強大是強大，可這苦逼的娃就像乾坤大挪移，有幾句話你就是身負九陽神功這種萬金油絕學的張無忌也練不成。相信放到性能更好的 *nix 系統下他的表現會好些，不過我現在是 Windows 用戶。

h2. 小無相功 EverEdit

原名 LotusEdit ，在我剛剛開始撰寫本文時，他剛剛改名不久，各方面都不大完善，但後來積極借鑒，越來越強。
軟件作者是中國人，軟件目前閉源但免費。作者自稱反思 EditPlus 而來，但我個人認爲主要借鑒對象是 EmEditor ， EmEditor 的主要功能他都有，圖標也很像。他主要抄了什麼功能呢？ Snippets, Project, Outline, 外部工具，批量搜索，任意字符高亮，但凡抄了的，相似度 90% 。在抄了很多功能的基礎上， EverEdit 仍然保持高性能和低佔用，作者的開發熱情很高，積極吸取其他軟件的優勢，所以這款算是 EmEditor 的優良替代品。特別值得一提的是，他的替換是我在 Windows 下試用過的文本編輯器裏最快的一個，我試過處理四十多萬行的大漢和辭典詞條，運用正則表達式的方式一次性做四十多萬個替換，十秒左右就完成了，比 EmEditor 快很多。
不過我要吐槽的是，這貨竟然沒有自動保存這麼基本的功能？另外右鍵無法直接刪除——他竟然把刪除做成一個子菜單，要等他彈出纔能操作。另外很奇怪的是，該軟件僅有綠色版，沒有安裝版，無法設置文件關聯。

h2. 玉女劍法 RJ TextEd

又一款開源編輯器。這個編輯器比較針對網頁開發，可以輕鬆的使用 Chrome, FF, IE 來預覽。有一個 html 菜單，功能上和別家的 html 工具欄相仿，但更強大，也更整齊。另外，他還可以輕鬆添加 CSS, JS, PHP 的常用語法。編輯器內置雙窗口文件管理器，比較方便進行文件移動、複製、刪除等操作。
比較有趣的一個功能是統計。一般文本編輯器的統計功能比較簡陋，不包含詞頻，這一款不但有詞頻，而且還給出百分比。不過這個功能是基於空格、換行、標點的，所以中文不好使——他的中文支持眞的有待提高，中英混排折行完全不行。
說一下缺點罷，速度較差，主題配置莫名其妙，最奇怪的是他的工具欄竟然沒有鼠標懸停提示，操作上和成熟編輯器也有一定差距。
也許他僅僅是還沒找到他的全眞劍法來雙劍合璧。

h2. 排座次

最後做一個列表來總結一下，去掉幾款個人覺得優勢不足的。這個評分除了「系統」一項針對 Linux 的完善度，其他都在討論 Windows 下的體驗。開源跨平臺軟件在 Linux 下的表現會比 Windows 下好。

基本上 5 分是個分水嶺，低於這個值我比較無法忍受。
如果要排梯隊， Emacs 必然是一梯隊； VIM 姑且給他一個一點五梯隊；接下來是 SciTE, jEdit, EmEditor 的二梯隊；然後是 EverEdit, AkelPad, EditPlus, UltraEdit 等構成的三梯隊，達不到三梯隊的譬如 Notepad++ 之流就算了。
本來還有兩個不錯的編輯器， Gedit 和 Kate ，不過二者都是藉助自身的桌面系統的，在 Linux 下的表現比 Windows 下好得多，放進來比較太不公平，且二者在功能方面沒有特別突出的表現，屬於比較中庸的產品，就不討論了。其他比如 X-ConTEXT 之類的半成品基本上可以忽略。

h2. 收式

文本編輯器和輸入法都是非常私人的東西，沒有絕對的好，但似乎有絕對的差。對我而言， EmEditor 和 EverEdit 是最舒適的， AkelPad 也不錯。如果使用 Linux ，我會比較願意使用 SciTE 和 jEdit 。目前暫時找不到一款讓我完全滿意並停止尋找之路的編輯器。
