對於經常訪問自己硬盤內各種內容的人來說，資源管理器類軟件真是一刻也離不開。但每個人對資源管理的方法不同，使用上也各有區別，故而在具體的軟件選擇上也頗有差異。但總有一些軟件，在他的領域上顯得比較出類拔萃，或是某些用法廣爲接受，這就比較值得討論了。
這類軟件數量極大，但歸納起來，他們的發展方向無非幾種：第一是界面增強， Windows Explorer 只有一個文件面板，我就給你變成兩個或更多，以便在不同的文件夾之間快速跳躍，或者是給界面加上標籤頁，即使只有一個，也能保持着很多個一直開着的狀態。第二是定位增強，比如把常用的文件加到軟件的收藏夾，或是自帶快速的搜索，再或者把文件打上標籤以便快速定位。第三是操作增強，比如增強快捷鍵，或是增強右鍵，再或者有一堆增強按鈕。第四是功能增強，比如這類軟件通常都有批量重命名功能和壓縮解壓功能。第五是網絡增強，比如加入 FTP, WebDAV ，或是同步到網盤、相冊等。第六是功能整合，一般通過插件或集成軟件的方式增加，這就需要有比較好的開放性了。
好的，就來討論一下這四個領域做得較好的若干軟件罷。

h2. Windows Explorer (WE)

自帶的東西，儘管不斷升級改造，但還是相當爛。微軟就是這樣，系統賣那麼貴，自帶的東西沒幾個好用（比如瀏覽器，比如文本編輯器，比如圖像查看器，比如多媒體播放器）。

h2. Total Commander (TC)

這款軟件在六個方面都做到了，加上軟件的速度相當快，故而很多用戶將他捧得上了天。既然大家捧，那就把他當標杆好了。
TC 是支持多標籤頁的雙面板，支持收藏夾和搜索，可以使用 Norton Commander 式快捷鍵以及自定義快捷鍵，可以鼓搗自定義按鈕，自帶功能豐富，支持 FTP （並且可以通過插件支持 WebDAV ），插件也相當多，與其他軟件的整合性也很好。
在我個人看來，他的最大亮點在於內置的查看功能指向文件夾的時候相當於查看屬性，而且查看大文件的時候速度很好。另外批量重命名功能在單一文件夾內的表現也非常好——強大而又非常便捷。缺點也有，比如他的功能太多導致重點太不突出，上手難度較大等。
總體來講， TC 的哲學是 Emacs 哲學去掉 free 的成分。強大是 TC 的最大目標，他就是個僞裝成資源管理器類軟件的操作系統。作者對收費也抓得不是特別嚴格，免費用戶只在開啓軟件的時候會遇到提示。

h2. Directory Opus (DO)

與 TC 齊名的軟件。我對其界面增強最爲滿意——左邊是文件樹，中間兩個面板，右邊一個查看窗口。
DO 的查看窗口雖然對文件夾是無效的，但能夠查看很多擴展名的內容，比如 zip ，他就直接顯示這個壓縮包裏有什麼（而非像 TC 那樣必須安裝插件，否則顯示亂碼），再如 html ，他會直接調用 IE 來查看。不過他對不同語系的支持並不好，比如 UTF-8 ，如果不帶簽名，在 DO 裏就是亂碼，而在 TC 裏是能自動檢測的。
DO 比起 TC ，更像是一個現代產品。比如 Vista 開始出現的「庫」功能， TC 似乎不屑一顧，而 DO 就增加了。而且 DO 的 UI 也看着比較舒服，是花了些功夫的。在跳轉方面， DO 也很跟得上時代，內置就有轉到開始菜單、轉到回收站、轉到庫之類的東西，讓你深切感受到你在用 Windows 。
但 DO 的優點大約也就這麼多了。他在功能增強方面顯得比較偏頗，該有的沒有（比如幾乎所有同類軟件都有的複製路徑功能），受衆極小的卻又有（比如同步到 Flickr 這個功能）。加上他的防盜版機制做到了極致，而價格又相當不菲，所以用戶量小於 TC 。

h2. Q-Dir

Q-Dir 是個非常簡潔的免費軟件，基本上他就是個支持一到四個面板的 WE 右鍵增強版。
由於他並不在 WE 的基礎上增加太多，用戶適應期必然很短，而右鍵增強的功能雖然很少，卻還實用。
儘管這款軟件單用略顯得單薄，與其他軟件配合倒是不錯。

h2. Clover

給 WE 加上標籤頁的免費軟件，跟 Q-Dir 正好是兩種不同的界面增強。

h2. Explorer++

一款在 WE 基礎上增加標籤頁並增強工具欄以及少量功能的開源軟件。軟件風格很 XP ，如果對這種風格很喜愛，或者乾脆你還在使用着 WE 的話，會覺得是一款很不錯的軟件。但功能上顯得有點單薄，加上雙面板就好了。

h2. Just Manager (JM)

新興的免費軟件，擁有 TC 的多數常用功能，而且非常小巧。雖然是新軟件，而且功能上已經不單薄了，但他的穩定性和速度都沒得說。 JM 最特色的是他的文件面板可以多達 16 個，是同類軟件中最多的。
我認爲他的開發到現在還只是一個階段，後續仍有更大發展。

h2. Tabbles

一款 freemium 軟件，跟 Vista 以來的庫很像，就是給文件打標籤，但可以打子標籤什麼。
軟件本身如果多加一個預覽窗口會不錯，不過遺憾的是沒有。而且便攜版竟然不支持不同盤符的文件，太狹隘了。

h2. XYplorer

一個樣樣通樣樣鬆的收費軟件。多面板是有，但也做不到極致。也可以設置預覽面板，但卻結合了 TC 和 DO 的缺點。亮點是在樣樣都有點的基礎上可以給文件打標籤。
軟件的響應速度不是很快，有點拖泥帶水的感覺。總之，軟件雖有值得借鑑的地方，卻並不好用。

h2. Double Commander (DC)

跟 JM 比起來， DC 在功能上略遜，勝在開源跨平臺罷。

h2. Multi Commander (MC)

細節做得非常用心的一款免費軟件。他在按鈕的設置方面做得非常用功。 TC 內部對其按鈕可添加的功能進行了簡單分組，而 MC 是三級分組，而且對腳本、外部命令的結合非常棒。如果有一定的編程基礎，會覺得這款軟件非常稱手。

h2. FreeCommander XE (FC)

專門和收費軟件搶飯碗的免費神器。在吸收好東西這方面，甚至強於收費軟件。
在面板上，他吸收了 DO 的優點，而在操作和預覽上，他又跟 TC 差不多，而且通殺 TC 的 wlx 插件。 FC 還具有非常好的收藏功能——倒不是說收藏本身有何創新，而是用戶可以用收藏樹來代替資源管理器式的文件樹。他的批量重命名雖然不及 TC 方便，卻很強大，支持直接用 MP3 標籤或是圖片像素等來命名，非常多媒體化。
FC 的模仿能力之強，幾乎是騰訊水準。不過當然也有創新——我很喜歡的分隔欄。一般此類軟件都有上層的工具欄和下層的按鈕欄，而 FC 還多了一個在兩個文件面板之間的分隔欄。功能上跟工具欄一樣，但位置格外好。如果是小屏幕或許還感覺不到其優勢，但對於比較大的屏幕，直接在兩個面板之間操作，可以減小眼球上下活動的範圍。而人眼的特點是水平視角寬闊、垂直視角狹窄（所以絕大多數文字是橫排的），這樣一個貼心的分隔欄實在是鼠標黨之福。
FC 最大的缺點是穩定性不佳，網上偶爾有人反映無法使用，我自己也覺得反應略慢——大概是慢了 100ms ，不是很明顯，但能感覺到。

h2. Unreal Commander (UC)

可以說他是一款很酷的免費軟件。爲什麼呢？因爲他的內置圖形主題很豐富且色彩比較 geek ，而且工具欄圖標也有好幾套不同風格可供選擇。感覺有點像文本編輯器裏的 ST2 。另外他對系統文件夾的歸納是諸多軟件中最爲詳盡的。其他方面則跟 TC 保持較高同質性，批量重命名窗口是那麼的像，而且 TC 的插件也可以直接拿來用。
不過，軟件雖然是免費的，卻還要授權，而且軟件經常彈窗求捐贈（果然是 ST2 風格）。便是這個原因罷，本款軟件是諸多免費軟件中唯一不原生提供便攜版的。

h2. 小結

此類軟件，對我來說，使用頻率不及文本編輯器，所以沒有太用心體驗。
