網絡本來自由度是很大的，但是我們總是面臨各種人為的限制。在家，你會被一個叫 netsniper 的東西限制使用路由器；在中國的學校，你會被一個叫教育網的東西限制訪問國外網站，在整個中國，你會被一個叫防火長城的東西限制訪問黨媽不喜歡的網站，甚至到了 NTU ，還有一個叫 CITS 的東西不許你用迅雷、優酷……
於是我來開一個介紹帖，分類討論各種類型的常用軟件，但不提供教學。由於翻越軟件極多，我不可能一一介紹，所以祇重點羅列一些業界內有名且目前較為有效的翻越方式。

h2. 準備

瀏覽器一般也要做點準備的，不過並不難。每個瀏覽器都要在代理服務器那裡設一下端口和地址，如果是 Chrome 系列，還可以用 Proxy Swichy! 插件來選擇性越獄；而 FireFox 也有 AutoProxy ； Opera 不用裝插件，但比較弱； IE 要自己寫腳本，再根據需要改腳本。

h2. 遠程 DNS

在眾多封鎖中，國防牆的威力第一，不但封了人家 IP ，還要污染 DNS ，就算你用了其他工具，能夠訪問那個被封鎖的 IP 了，還是會因為 DNS 被污染而無法正確解析。這一招其實挺狠的，直接廢掉域名啊。

h3. SocksCap 和 FreeCap

這是遠程解析軟件。

h3. FireFox

在 about:config 修改 network.proxy.socks\_remote\_dns 為 true 。

h3. Chrome

最新版的可以了。

h3. OpenDNS 和 Google DNS

在 TCP/IP 協議填寫他們提供的 IP 即可。嫌麻煩可以用 Google DNS Helper 。

h2. Hosts 文件法

很多網站的 IPv6 沒被封，只要修改 hosts 文件，再裝個六飛或 gogoClient ，就可以直接訪問了。不過我相信這個方法用不久了。有的網站僅僅被污染了 DNS ，在 hosts 文件中設置正確 IP 地址也可以解決問題。這個系列的方法只解決了一部分問題，但是速度很好，因為你是直接訪問被屏蔽網站的。但是局限性很強，比如關鍵字屏蔽的作用，比如牆 IP 的問題。

h3. HostsX

一款記事本風格的免費軟件，擁有少量網站的 IP 。

h3. SmartHosts

一個安置在 Google Code 的雲 hosts 文件，比較針對手機用戶，而且網站不多。

h2. https 法（或動態 SSL 法）

https 是加密瀏覽，所以往往不受牆的控制，也是穿牆，而且使用方法是所有方法中最最簡單的：把網址前頭的 http 改成 https 就可以了。 Dropbox 和 Facebook 用這個方法都是可行的，另外 Gmail 也建議使用。此法宜配合 Host 文件進行，方能發揮最大功效。
於是 Chrome 的優勢就出來了：可以強行使用 https 訪問網站。方法是 chrome://net-internals — HSTS — Add domain
以上方法除了遠程 DNS 都對於外界的依賴性不大，不用考慮付費、代理服務器被封的問題，管好你的電腦就是了。雖然具有局限性，不過祇要在 hosts 文件裡有了正確的 IP 地址，並強制使用 https ，你很少能感覺到牆的存在……

h2. 在綫代理

h3. 免費綫代理

這是什麼技術都不需要掌握，什麼準備都不用做，什麼軟件都不用下載的方法，但是速度很慢，一般都是教育網用用而已。

h3. 自架在綫代理

如果有境外虛擬空間，可以通過自行搭建在綫代理的方式翻越，比如使用 APJP, phpsocks5, phproxy encrypt, KnProxy, PHP Tunnel Proxy, hyk-proxy 之類的程序。好處是比起人滿為患的公共在綫代理，你自己的空間的速度總是好的。這種方法當然需要學會用用 FTP ，改改參數什麼的，不過還是很容易。其中， phpsocks5 最為安全，而 APJP 則可以添加多個代理。
在綫代理還有一個優點，那就是祇要有瀏覽器就能上，不依賴平臺。

h2. 郵箱法

有點像前者，但也不盡然。向往某個特定的郵箱發一封郵件，在郵件的標題或正文包含你要訪問的 URL ，該郵箱會自動回信，信上就是訪問內容。某些郵箱甚至還可以留言。不過這種方法蠻蛋疼的……

h2. VPN

這個系列適合小白使用，也是網上比較流行的手段。我個人並不喜歡，因為他是全局翻越，而我瀏覽的網站多數沒有被禁止訪問。很多賣 VPN 的都會提供客戶端，所以比較不必擔心。

h3. 自架開源 VPN

如果有 VPS ，可以用 L2TP, PPTP 之類的程序自行架設 VPN 。不過我覺得有 VPS 的話用 SSH 更好……

h3. 輪子系列

輪子的門可以在中國和伊朗使用，不需要自己找帳號就能上。輪子還有很多產品，但是還是門最廣泛使用。新版的門已經脫離傳統模式，轉而成為後面將提到的「加速器」模式，可以實現選擇性跨越和全程加密，非常高級，我自己也常在深夜使用，並且非常鼓勵小白用。由於免費且用戶極多，輪子系列速度和穩定性沒有太多保障，加上樹大招風，是上國的重點打擊對象，有時會不好使。無論如何，輪子系列以其免費、易用一直佔據著原貌大作戰中非常重要的一席。

h3. chnroutes

用他來設置你的 VPN ，可以實現自動區分國內外服務器並選擇性翻牆，這是 VPN 的最高形態。

h2. SSH

這系列速度和穩定性都不錯，自己定制也比較方便。找帶 SSH 的免費 CPanel 空間是個不錯的選項。

h3. MyEnTunnel

非常小，但密碼保護得相當完好，挺不錯。但是，原裝的 plink 是限速的，需要自行以不限速的 plink 來替代。

h3. Bitvise Tunnelier

一個拿來玩 VPS 的軟件，能當 FTP 軟件來用，還可以 SSH 遠程控制。

h3. PuTTY + plink

PuTTY 的主要功能其實不是代理，而是在瘟到死系統下遠程命令行控制 Linux 系統的電腦（圖形控制用 XManager ），不過你要用來翻越當然也很好。

h2. 加速器

現在翻越的人多了，於是有人想出來專門做這個生意。 VPN 太不爽了， SSH 好是好，就是價格不便宜且服務器未必適合自己，於是有了加速器。
我用過一款，同一帳號可以選擇歐美十幾個服務器，而且速度都非常快，比我家網速快多了。所以如果你願意花一些錢，一款優質的加速器絕對是好選擇。
加速器的原理各不相同，有人甚至想出來用 GAE 來做（ EZProxy ），但對用戶來說都是差不多的東西……
從在綫代理到郵件到 VPN 到 SSH 再到加速器，其實原理都是一樣的，就是在境外找一台沒有被屏蔽的服務器，讓他把你要瀏覽的網頁直接傳給你。

h2. GAE 系列

這一系列利用 Google 的 GAE ，說白了還是境外服務器，不過這次用的是 Google 的。由於 Google 的服務器帶寬充足， 1080p YouTube 觀看無壓力（當然你家帶寬得夠），而且 GAE 在流量不大時是免費的，適合家庭用戶。目前， GAE 系列是最受中高端用戶青睞的一個系列，這得多謝股溝提供如此穩定的免費服務。最近方校長對 GAE 非常關照，需要 hosts 修改進行。

h3. GAppProxy

先註冊一個 Google App Engine,?再下好所需的 <a href="http://www.python.org/ftp/python/2.5.1/" rel="external">Python</a>, <a href="http://code.google.com/appengine/downloads.html" rel="external">App Engine SDK </a> 。最後到 <a href="http://gappproxy.googlecode.com/" rel="external">GAppProxy</a> 的 download 部分下載其說明與所需文件，並按照說明一步一步做完，就搞定了~~缺點是完全沒有加密。作為 GAE 翻越界鼻祖，他是功不可沒的。

h3. GoAgent

把 GAppProxy 便捷性推向高峰的一款力作， python 環境和 GAE 那傻叉的工具都不用裝，而且可以開幾個 GAE 同時工作。另外此軟件是跨平臺使用的， Windows, MAC, Linux, Android, iOS 均可。另外如果你有境外 PHP 空間，也可以通過他來安裝獨享的代理。

h3. Secure-GAppProxy

解決了 GAppProxy 安全性不足的問題，具有圖形介面。

h3. hyk-proxy

和 GoAgent 一樣，都是兩棲的，可以在免費的 SeattleGENI （註冊需要 edu 郵箱）上運行。

h3. Snova

支持多種免費 PaaS 的一款工具。雖然 GAE 系列免費方便速度快，但樹大招風，有時不免要被牆一下，這時 Snova 的兼容性就派上用場了。

h2. Telex

還是基於代理服務器思維，但用上了 https 技術的新工具。
牆是關鍵字屏蔽的，但 https 是 64 位加密，加密後牆會無法得知我們瀏覽的具體內容。 Telex 會在一個正常訪問的網站請求上附加一個加密的標籤，標籤就是那個被屏蔽的網站。
這玩意是密歇根大學的發明，嗯，帝國主義亡「我」之心不死啊。不過其官方網站也承認這更多的是利用

h2. P2P 型

這次不是「境外服務器」了，而是「境外用戶的電腦」。
本來很多工具是用來匿名訪問的（匿名訪問對黑客來說是基本素質，因為一旦你的 IP 暴露，你就離局子不遠了，很多黑客喜歡先侵入某個服務器，然後在用該服務器做跳板，跳板甚至還不止一個），悲催的國人卻要拿來翻越。
我個人並不使用這種方式，因為我家帶寬非常有限，而我需要日夜不停的上傳數據……但無論如何， P2P 是真正無法完全封殺的方式。

h3. Tor

Tor 是一個 P2P 的代理，就像電騾下載一樣，祇不過傳輸的不是文件，而是網頁內容。由於被國內屏蔽中央目錄服務器（類似電騾的服務器），並不十分好用。

h3. I2P

可說是 Tor 的進化版。 Tor 每次都需要連接中央目錄服務器以便接上其他用戶，而 I2P 祇需連接上第一次即可，以後都使用 Kad 連接。
I2P 是強大而複雜的，而且可以在類 Unix 系統上使用，但是其官網毫不例外的處於高牆之外。

h3. GPass+Skype

Skype 是一個 P2P 軟件，而 GPass 能夠幫你利用 Skype 的 P2P 功能傳輸信息。當然了，這對對方的要求很高，如果對方網絡不好，速度就不好。

h2. 西廂

西廂系列有兩季，二者毫無關係，卻用著同一名字。他們的厲害之處在於穿越而非翻越——他們是利用牆的漏洞來直接穿牆而過，所以速度相當於直接連接的速度，這是其他方法無法全面實現的。我個人認為這個名字起得不好，因為張生是翻過去的，不是穿過去的，應該叫「嶗山計畫」，典故是聊齋志異裡能穿牆而過的嶗山道士。
西廂祇能在 Linux 下使用。

h2. 專職瀏覽器

一般是基於某款著名瀏覽器打包而成的，好處是非常適合小白。一般是 SSH 銷售商為了推銷自己的 SSH 帳號而打包設計的，他們的帳號一般都存在少量超售的問題，但軟件本身是很好的。其實我也有自己的專制瀏覽器，那就是強制使用 https 的 Chrome ，安裝了 SwitchySharp （當然還有無關插件，如 IE-Tab, Ultimate Chrome Flag, LastPass, Cookie Swither, Facebook Like Button）

h3. vforchrome

基於 Chromium ，在扳手那裡填入自己的 SSH 帳號即可。

h3. SSHWall

基於 FireFox ，本身提供一個免費帳號，這個號高峰期有成千上萬的人在用，但如果避開繁忙時段，可以達到 100KB/s 左右。他集成的插件是 AutoProxy ，軟件是 MyEnTunnel + 不限速的 Plink 。

h3. TorChrome Portable, TorFireFox Portable, TorOpera Portable, Twitter Browser Portable

就是基於 Chrome/FireFox/Opera ，使用了 Tor 的瀏覽器。最後一個是第一個的進化版，為的是不讓自己登入 IP 過於頻繁更換導致帳號被封鎖。

h2. 小結

首先，在中國，修改 hosts 文件是必須的。人民群眾也應該具有使用 https 的意識，以保障信息安全。
至於具體的翻越方法，比較好用的當然是購買加速器和 SSH 帳號，而免費的 GAE 以及免空的 SSH 帳號也不失為優質選項。
原則上不建議僅僅掌握過於大眾化的翻越方式（如輪子的工具），原因是大眾化的方式容易被封鎖，而你一旦受困於高牆又沒掌握其他方式，那就非常不幸了。所以建議下載多幾個工具，一個不行換一個。
如果以上方法全部失效，我還有最後一招： QQ 聯繫國外親友，然後要求遠程控制。當然這個方法是愚蠢而無聊的。
